一、問答題（20%）

  1. 從 path length和clustering coefficient的觀點，說明為何人際關係是具有小世界特性的網絡結構？(10%)[10%]
人際關係通常滿足群聚係數(clustering coefficient)高，即常常存在同溫層，但人與人的最短距離(path length)短，如六度分隔的實驗結果所示。有一部分少數人擁有眾多朋友，能跨過群集之間的連結，使節點之間的距離縮短。因此人際關係是具有小世界特性的網絡結構。
  1. 說明HITS演算法計算網絡節點的authority與hub指標數值的意義，並以「好友提名的友誼網絡（例如：由班上同學以提名法，填答同班級的好友名單）」為例，說明這兩個指標的差異。[10%]
權威性(authority)與樞紐性(hub)，權威性從多少(重要)節點連結到自己來估計中心程度，樞紐性估計連結出去到其他(重要)節點的程度。且高樞紐性節點指向高權威性節點，高權威性節點是由高樞紐性節點所指向的。
從好友提名的友誼網絡中，權威性高，代表越多同學認為這個人是他的朋友（表示這個人是受歡迎的）；樞紐性高，代表他認定的朋友中，有許多權威性高的人（表示這個人的朋友多是受歡迎，認定性高、很有眼光）。

  1. 從結構洞指標Efficiency的觀點，比較 node 2和 node 5的個體網絡（ego-network）能從網絡獲取的競爭力。[計算Efficiency 5% + 說明5%]
Efficiency = Effective Size ÷ ego網絡中alters節點數量
node5的egos網絡，alters節點數量=4(Va)，alters彼此的連線數量=3(Ea)。
Effective Size (node 5) = Va-Ea×2÷Va=(4)-(3)×2÷(4)=2.5
Efficiency (node 2) = 2.6/5 = 0.52
Efficiency (node 5) = 2.5/4 = 0.625
Node 5 的競爭力較大。雖然Node 2有較多連結，但資訊的來源重複，或有冗餘，且 alters 之間的都有連結；Node 5則為兩個群體之間的橋樑，雖然連結節點少，但能掌握多方訊息，在網絡中存在有結構洞。

3.針對Node 34 (“John A”) 為ego的個體網絡，以節點屬性age (C: child; A: adult, E: elder)，用 E-I index 比較中心節點(ego)和alters的屬性相似性(homophily)，並將計算結果的數值進行說明。[10%]

Ge=egos[[34]]
ext=sum(V(Ge)$age!=V(Ge)["John A"]$age)
int=sum(V(Ge)$age==V(Ge)["John A"]$age)-1
EI=(ext-int)/(ext+int)
sprintf("John A個體網絡的中心節點E-I指數 = %.4f",EI)
[1] "John A個體網絡的中心節點E-I指數 = 0.1765"
V(Ge)$name=gsub("Actor ","",V(Ge)$name)
plot(Ge,vertex.color=factor(V(Ge)$age),vertex.size=20,layout=layout_as_star(Ge,V(Ge)["John A"]))

相連節點中，與中心網絡age性質不同的節點數：10
相連節點中，與中心網絡age性質相同的節點數：7
E-I指數(針對節點)=(10-7)/(10+7)=3/17=0.1765
E-I指數大於0，表示周圍節點與中心節點的屬性多為不同。
E-I指數越大（正數，越接近1），同質性(homophily)越低；E-I指數越小（負數，越接近-1），同質性(homophily)越高。
John A個體網絡的中心節點E-I指數=0.1765，表示周圍節點的年齡層與John A自己較不像。
